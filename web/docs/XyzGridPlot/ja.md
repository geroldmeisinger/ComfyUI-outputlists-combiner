## XYZ-GridPlot

![XYZ-GridPlot](XyzGridPlot/XyzGridPlot.png)

(ComfyUI workflow included)

画像のリストからXYZ-Gridplotを生成します。
これは画像のリスト（バッチを含む）を取り、最初にそれらを長いリストにフラット化します（したがって`batch_size=1`）。

**グリッドの形状**
グリッドの形状を決定します：
1. 行ラベルの数
2. 列ラベルの数
3. 残りのサブイメージ。
`order=inside_out`を使用してイメージ選択を逆転させることができます（`batch_size>1`の場合に便利で、バッチにラベルを付ける場合）。

**配置**
* ラベルが次の行に折り返されると、全体の軸は「マルチライン」として扱われ、均等な間隔で上に配置されます。
* すべてのラベルが数字であるか、すべてのラベルが数字で終わる場合（例：`strength: 1.`）は、全体の軸は「数値」として扱われ、右に配置されます。
* その他のすべてのテキストは「シングルライン」として扱われ、中央揃えになります。
* 列のシングルラインと数値ラベルは下に配置され、行のラベルは垂直方向に中央揃えになります。

**フォントサイズ**
* 列ラベル領域の高さは`font_size`または各行の最大サブイメージパッキング高さの半分（どちらか大きい方）で決定されます。
* 行ラベル領域の幅はサブイメージパッキングの最大幅で決定されます（最小256px）。
* テキストは収まるまで縮小され（最小`font_size_min=6`）、全体の軸（行ラベルまたは列ラベル）で同じフォントサイズが使用されます。
フォントサイズが既に最小値の場合、残りのテキストは切り取られます。

**サブイメージパッキング**
サブイメージ（通常はバッチから）を最も正方形の領域（「サブイメージパッキング」）に整形します。ただし`output_is_list=True`の場合は、各セルに1つのイメージのみを使用し、全体のイメージグリッドのリストを作成します。
このイメージグリッドのリストは、別のXyzGridPlotノードに接続してスーパーグリッドを作成するために使用できます。
サブイメージが異なるサイズのバッチで構成されている場合、不足するセルは空のイメージで埋められます。
セルあたりのイメージ数（バッチイメージを含む）は`rows * columns`の倍数である必要があります。

### 入力

| 名前 | タイプ | 説明 |
| --- | --- | --- |
| `images` | `IMAGE` | 画像のリスト（バッチを含む） |
| `row_labels` | `*` | 左側の行ラベルテキスト |
| `col_labels` | `*` | 上部の列ラベルテキスト |
| `gap` | `INT` | サブイメージパッキング間のギャップ。サブイメージ自身の間にはギャップがありません。サブイメージ間のギャップが必要な場合は、別のXyzGridPlotノードを接続してください。 |
| `font_size` | `FLOAT` | 目標フォントサイズ。テキストは収まるまで縮小されます（最小`font_size_min=6`）。 |
| `row_label_orientation` | `COMBO` | 行ラベルのテキスト方向。スペースを節約したい場合に便利です。 |
| `order` | `BOOLEAN` | イメージを処理する順序を定義します。これはサブイメージがある場合にのみ関係します。`batch_size>1`でバッチをプロットしたい場合に便利です。 |
| `output_is_list` | `BOOLEAN` | これはサブイメージがある場合、またはスーパーグリッドを作成したい場合にのみ関係します。 |

### 出力

| 名前 | タイプ | 説明 |
