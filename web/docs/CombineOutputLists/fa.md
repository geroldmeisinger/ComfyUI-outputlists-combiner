## ترکیب خروجی‌لیست‌ها

![ترکیب خروجی‌لیست‌ها](CombineOutputLists/CombineOutputLists.png)

(فایل کارگاه ComfyUI همراه داده شده است)

تا ۴ خروجی‌لیست را می‌گیرد و تمام ترکیب‌های آن‌ها را تولید می‌کند.

مثال: `[1, 2, 3] x ["A", "B"] = [(1, "A"), (1, "B"), (2, "A"), (2, "B"), (3, "A"), (3, "B")]`

`unzip_a` .. `unzip_d` از `is_output_list=True` (که با نماد `𝌠` نشان داده شده است) استفاده می‌کنند و به صورت متوالی توسط گره‌های مربوطه پردازش می‌شوند.

همه لیست‌ها اختیاری هستند و لیست‌های خالی نادیده گرفته می‌شوند.

به صورت فنی این کار *ضرب دکارتی* را محاسبه کرده و هر ترکیب را تقسیم‌بندی شده به عنوان عناصر جداگانه خروجی می‌دهد (`unzip`)، در حالی که لیست‌های خالی با واحدهای `None` جایگزین می‌شوند و آن‌ها `None` را روی خروجی مربوطه منتشر می‌کنند.

مثال: `[1, 2] x [] x ["A", "B"] x [] = [(1, None, "A", None), (1, None, "B", None), (2, None, "A", None), (2, None, "B", None)]`

### ورودی‌ها

| نام | نوع | توضیح |
| --- | --- | --- |
| `list_a` | `*` | (اختیاری) |
| `list_b` | `*` | (اختیاری) |
| `list_c` | `*` | (اختیاری) |
| `list_d` | `*` | (اختیاری) |

### خروجی‌ها

| نام | نوع | توضیح |
| --- | --- | --- |
| `unzip_a` | `* 𝌠` | مقدار ترکیب‌های مربوط به `list_a`. |
| `unzip_b` | `* 𝌠` | مقدار ترکیب‌های مربوط به `list_b`. |
| `unzip_c` | `* 𝌠` | مقدار ترکیب‌های مربوط به `list_c`. |
| `unzip_d` | `* 𝌠` | مقدار ترکیب‌های مربوط به `list_d`. |
| `index` | `INT 𝌠` | محدوده 0..count که می‌تواند به عنوان شاخص استفاده شود. |
| `count` | `INT` | کل تعداد ترکیب‌ها. |
