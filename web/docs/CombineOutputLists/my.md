## OutputLists အထဲမှ ပေါင်းစပ်မှုများ

![OutputLists Combinations](CombineOutputLists/CombineOutputLists.png)

(ComfyUI workflow ပါဝင်ပါသည်)

OutputLists 4 ခုအထိကို ယူပြီး အားလုံး၏ ပေါင်းစပ်မှုများကို ဖန်တီးပါသည်။

ဥပမာ: `[1, 2, 3] x ["A", "B"] = [(1, "A"), (1, "B"), (2, "A"), (2, "B"), (3, "A"), (3, "B")]`

`unzip_a` .. `unzip_d` များသည် `is_output_list=True` ကို အသုံးပြုပြီး (symbol `𝌠` ဖြင့် ပြထားပါသည်) အိုင်းနှစ်သော အလုပ်များကို အတိုင်းအတာမှာ ပြုလုပ်ပါလိမ့်မည်။

စာရင်းအားလုံးသည် မဖြစ်မီ ဖြစ်နိုင်သောအရာများဖြစ်ပြီး ကွာလွန်သောစာရင်းများကို မလုပ်ကြောင်း မှတ်သားပါလိမ့်မည်။

တည်းဖြတ်ပြုလုပ်မှုအားဖြင့် သူတို့သည် *Cartesian product* ကို တွက်ချက်ပြီး အားလုံး၏ ပေါင်းစပ်မှုများကို အစိတ်အပိုင်းအတိုင်း ပြသပါလိမ့်မည် (`unzip` ဖြင့်) သို့မဟုတ် ကွာလွန်သောစာရင်းများသည် `None` တွင် အစိတ်အပိုင်းများဖြင့် အစားထိုးပြီး အိုင်းနှစ်သော ထွက်လာမှုများတွင် `None` ကို ထုတ်ပေးပါလိမ့်မည်။

ဥပမာ: `[1, 2] x [] x ["A", "B"] x [] = [(1, None, "A", None), (1, None, "B", None), (2, None, "A", None), (2, None, "B", None)]`

### ထွက်လာမှုများ

| အမည် | အမျိုးအစား | ဖော်ပြချက် |
| --- | --- | --- |
| `unzip_a` | `* 𝌠` | `list_a` နှင့် ပေါင်းစပ်မှုများအတွက် တန်ဖိုးများ။ |
| `unzip_b` | `* 𝌠` | `list_b` နှင့် ပေါင်းစပ်မှုများအတွက် တန်ဖိုးများ။ |
| `unzip_c` | `* 𝌠` | `list_c` နှင့် ပေါင်းစပ်မှုများအတွက် တန်ဖိုးများ။ |
| `unzip_d` | `* 𝌠` | `list_d` နှင့် ပေါင်းစပ်မှုများအတွက် တန်ဖိုးများ။ |
| `index` | `INT 𝌠` | 0..count အတွင်းရှိ အညွှန်းအမှတ်ဖြစ်ပြီး အညွှန်းအမှတ်အဖြစ် အသုံးပြုနိုင်ပါသည်။ |
| `count` | `INT` | စုစုပေါင်း ပေါင်းစပ်မှုအရေအတွက်။ |
