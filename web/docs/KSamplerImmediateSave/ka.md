## KSampler Immediate Save

![KSampler Immediate Save](KSamplerImmediateSave/KSamplerImmediateSave.png)

(ComfyUI სამუშაო მოდელი ჩართულია)

ნაგულისხმევი `CheckpointLoader`, `KSampler`, `VAE Decode` და `Save Image` ნოდების გაფართოება, რომ ერთად დაამუშაონ.
ეს სასარგებლოა, თუ გსურთ არამომდევროვანი სურათების გადატვირთვა ბლოკებში უშუალოდ.

*"სურათის შესანახად მხოლოდ კოსტიუმური KSampler? ახლა მე გახდი იმ რამის მსგავსები, რასაც მინდოდა მოვართეთ!"*

### შეყვანები

| სახელი | ტიპი | აღწერა |
| --- | --- | --- |
| `cpkt_name` | `COMBO` | ჩამოსატვირთი მოდელის (ჩეკპოინტის) სახელი. |
| `positive` | `STRING` | პირობა, რომელიც აღწერს ატრიბუტებს, რომლებიც უნდა შეიცავდეს სურათში. |
| `negative` | `STRING` | პირობა, რომელიც აღწერს ატრიბუტებს, რომლებიც უნდა არ შეიცავდეს სურათში. |
| `latent_image` | `LATENT` | ლატენტური სურათი, რომელიც უნდა განახლდეს. |
| `seed` | `INT` | შემთხვევითი მნიშვნელობა, რომელიც გამოიყენება ხარისხის შესაქმნელად. |
| `steps` | `INT` | ნამატის რიცხვი დამალული პროცესში. |
| `cfg` | `FLOAT` | კლასიფიკატორის გარეშე მიმართულების მასშტაბი არჩევანის და შედეგების შორის ბალანსის მითითება. უფრო მაღალი მნიშვნელობები მოდის უფრო მიმართული სურათებში, თუმცა ძალიან მაღალი მნიშვნელობები უარყოფითად ახდენენ ხარისხის მოვლენას. |
| `sampler_name` | `COMBO` | სამომხმარებლო ალგორითმი, რომელიც გამოიყენება სამომხმარებლო მონაცემების მიღებისას. შეიძლება გავლენა მოახდინოს ხარისხზე, სიჩქარეზე და სტილზე. |
| `scheduler` | `COMBO` | განაწილების სისტემა აკონტროლებს ხარისხის მონაცემების მოხსნას სურათის შექმნამდე. |
| `denoise` | `FLOAT` | დამალული პროცესის რაოდენობა, ნაკლები მნიშვნელობები შეინახავს საწყისი სურათის სტრუქტურას, რაც უზრუნველყოფს სურათის სურათის მიმდებარეობას. |
| `filename_prefix` | `STRING` | ფაილის სახელის პრეფიქსი. შეიძლება მოიცავდეს ფორმატირების ინფორმაციას, მაგალითად %date :yyyy-MM-dd% ან %Empty Latent Image.width% სადაც მონაცემები არის ნოდებისგან მიღებული. |
