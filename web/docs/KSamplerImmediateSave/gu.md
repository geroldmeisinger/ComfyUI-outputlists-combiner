## KSampler Immediate Save

![KSampler Immediate Save](KSamplerImmediateSave/KSamplerImmediateSave.png)

(ComfyUI વર્કફ્લો સમાવેશ કરે છે)

મૂળના `CheckpointLoader`, `KSampler`, `VAE Decode` અને `Save Image` નો નોડ વિસ્તારણ જે એક સાથે પ્રક્રિયા કરે છે.
આ ઉપયોગી છે જો તમે જી માટે અંતર્મુખ ઇમેજ્સ સેવ કરવાની ઇચ્છા રાખો.

*"એક કસ્ટમ KSampler જે મૂળ ઇમેજ સેવ કરવા માટે? હવે મેં તે વસ્તુ બનાવી જે મેં નાશ કરવાની શોધ કરી હતી!"*

### ઇનપુટ્સ

| નામ | પ્રકાર | વર્ણન |
| --- | --- | --- |
| `cpkt_name` | `COMBO` | લોડ કરવા માટે ચેકપોઇન્ટ (મોડેલ) નું નામ. |
| `positive` | `STRING` | સંશોધન જે ઇમેજમાં સમાવવાની ઇચ્છા રાખો છો તેનું વર્ણન કરે છે. |
| `negative` | `STRING` | સંશોધન જે ઇમેજમાં અલગ રાખવાની ઇચ્છા રાખો છો તેનું વર્ણન કરે છે. |
| `latent_image` | `LATENT` | ડિનોઇઝ કરવા માટે લેટેન્ટ ઇમેજ. |
| `seed` | `INT` | શોધન માટે વપરાતી રૅન્ડમ સીડ. |
| `steps` | `INT` | ડિનોઇઝિંગ પ્રક્રિયામાં વપરાતી સ્ટેપ્સની સંખ્યા. |
| `cfg` | `FLOAT` | ક્લાસિફિયર-ફ્રી ગાઇડન્સ સ્કેલ રચના અને પ્રોમ્પ્ટની અનુસરણ વચ્ચે સંતુલન રાખે છે. ઉચ્ચ મૂલ્યો પ્રોમ્પ્ટ સાથે વધુ મેળવાયેલ ઇમેજ આપે છે પરંતુ ખૂબ ઉચ્ચ મૂલ્યો ગુણવત્તા પર નકારાત્મક અસર કરી શકે છે. |
| `sampler_name` | `COMBO` | નમૂના લેવા માટે વપરાતી એલ્ગોરિધમ, આ ગુણવત્તા, ઝડપ અને ઉત્પાદિત આઉટપુટની શૈલીને પ્રભાવિત કરી શકે છે. |
| `scheduler` | `COMBO` | સ્કેજ્યુલર શોધન માટે શોધન સ્થાનાંને ધીમે ધીમે દૂર કરીને ઇમેજ બનાવે છે. |
| `denoise` | `FLOAT` | લાગુ થયેલ ડિનોઇઝિંગની માત્રા, નીચી મૂલ્યો મૂળ ઇમેજની રચનાને રાખે છે જે ઇમેજ થી ઇમેજ નમૂનાને અનુમતિ આપે છે. |
| `filename_prefix` | `STRING` | સેવ કરવા માટે ફાઇલ માટે પ્રિફીક્સ. આ ફોર્મેટેટિંગ માહિતી સમાવી શકે છે જે જેમ %date :yyyy-MM-dd% અથવા %Empty Latent Image.width% છે જે નોડ્સમાંથી મૂલ્યો સમાવે છે. |

### આઉટપુટ્સ

| નામ | પ્રકાર | વર્ણન |
| --- | --- | --- |
| `image` | `IMAGE` | ડીકોડ કરેલ ઇમેજ. |
