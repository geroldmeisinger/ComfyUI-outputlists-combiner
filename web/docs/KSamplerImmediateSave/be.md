## KSampler Immediate Save

![KSampler Immediate Save](KSamplerImmediateSave/KSamplerImmediateSave.png)

(Уключаны ComfyUI workflow)

Расшырэнне вузла стандартнага `CheckpointLoader`, `KSampler`, `VAE Decode` і `Save Image` для апрацоўкі як аднаго.
Гэта карысна, калі вы хочаце захаваць промежкавыя выявы для сетак непасрэдна.

*"Кастомны KSampler толькі для захавання выявы? Цяпер я стаў тым, чым шукаў, каб знішчыць!"*

### Уваходы

| Назва | Тып | Апісанне |
| --- | --- | --- |
| `cpkt_name` | `COMBO` | Назва чакпойнта (мадэлі) для загрузкі. |
| `positive` | `STRING` | Умовы, якія апісваюць атрыбуты, якія вы хочаце ўключыць у выяву. |
| `negative` | `STRING` | Умовы, якія апісваюць атрыбуты, якія вы хочаце выключыць з выявы. |
| `latent_image` | `LATENT` | Латэнтная выява для дэшуміравання. |
| `seed` | `INT` | Выпадковы насінне, якое выкарыстоўваецца для стварэння шуму. |
| `steps` | `INT` | Колькасць крокаў, якія выкарыстоўваюцца ў працэсе дэшуміравання. |
| `cfg` | `FLOAT` | Маштаб бяспрэдметнага кіравання балансуе крэатыўнасцю і прытрыманнем да памысла. Больш высокія значэнні прыводзяць да выяў, якія больш чысцей супадаюць з памыслам, але занадта высокія значэнні негатыўна ўплываюць на якасць. |
| `sampler_name` | `COMBO` | Алгарытм, які выкарыстоўваецца пры аднаўленні, гэта можа ўплываць на якасць, хуткасць і стыль генераванага выніку. |
| `scheduler` | `COMBO` | Планавальнік кіруе тым, як шум паступова выдаляецца, каб утварыць выяву. |
| `denoise` | `FLOAT` | Колькасць дэшуміравання, якое прымяняецца, меншыя значэнні захоўваюць структуру пачатковай выявы, дазваляючы здзейсняць здымак выявы на выяву. |
| `filename_prefix` | `STRING` | Прэфікс для файла, які трэба захаваць. Гэта можа ўключаць інфармацыю фарматавання, такую як %date :yyyy-MM-dd% або %Empty Latent Image.width%, каб уключыць значэнні з вузлоў. |

### Выхады

| Назва | Тып | Апісанне |
| --- | --- | --- |
| `image` | `IMAGE` | Дэкадаваная выява. |
