## שמירת KSampler מיידית

![KSampler Immediate Save](KSamplerImmediateSave/KSamplerImmediateSave.png)

(הזרימה ב-ComfyUI נכללת)

הרחבת צומת ברירת המחדל `CheckpointLoader`, `KSampler`, `VAE Decode` ו-`Save Image` לעיבוד כ-unit אחד.
שימושי אם אתה רוצה לשמור את תמונות הביניים עבור רשתות מיידית.

*"צומת KSampler מותאם רק כדי לשמור תמונה? עכשיו הפכתי לזה בדיוק מה שחיפשתי להרוס!"*

### קלטים

| שם | סוג | תיאור |
| --- | --- | --- |
| `cpkt_name` | `COMBO` | שם הנקודות של הבדיקה (מודל) לטעינה. |
| `positive` | `STRING` | התנאים שמתארים את התכונות שברצונך לכלול בתמונה. |
| `negative` | `STRING` | התנאים שמתארים את התכונות שברצונך להשמיט מה תמונה. |
| `latent_image` | `LATENT` | התמונה הלטנטית לניקוי. |
| `seed` | `INT` | הזרם האקראי המשמש ליצירת הצלילים. |
| `steps` | `INT` | מספר הצעדים המשמשים בתהליך הניקוי. |
| `cfg` | `FLOAT` | סולם הנחיתות של הנחיית המודלים ללא מודלים מאוזן בין יצירתיות ואמינות לתשובה. ערכים גבוהים מובילים לתמונות תואמות יותר לתשובה עם זאת ערכים גבוהים מדי יפגעו באיכות. |
| `sampler_name` | `COMBO` | האלגוריתם המשמש כאשר מתבצע דגימה, זה יכול להשפיע על האיכות, המהירות, והסגנון של הפלט שנוצר. |
| `scheduler` | `COMBO` | מתזמן שולט איך הצלילים מוסרים בהדרגה כדי ליצור את התמונה. |
| `denoise` | `FLOAT` | כמות הניקוי המופעלת, ערכים נמוכים יותר ישמרו על מבנה התמונה הראשונית ויאפשרו דגימה מ תמונה לתמונה. |
| `filename_prefix` | `STRING` | הקידומת של הקובץ לשמירה. זה יכול לכלול מידע עיצובי כמו %date:yyyy-MM-dd% או %Empty Latent Image.width% כדי לכלול ערכים מצומת. |

### פלטים

| שם | סוג | תיאור |
| --- | --- | --- |
| `image` | `IMAGE` | התמונה המפועelsonת. |
