## KSampler негайне збереження

![KSampler негайне збереження](KSamplerImmediateSave/KSamplerImmediateSave.png)

(Включено ComfyUI workflow)

Розширення вузла стандартних `CheckpointLoader`, `KSampler`, `VAE Decode` та `Save Image` для обробки як одного.
Це корисно, якщо ви хочете негайно зберегти проміжні зображення для сіток.

*"Власний KSampler лише для збереження зображення? Тепер я став тим, чого хотів уникнути!"*

### Вхідні дані

| Назва | Тип | Опис |
| --- | --- | --- |
| `cpkt_name` | `COMBO` | Назва чекпоінта (моделі) для завантаження. |
| `positive` | `STRING` | Умови, що описують атрибути, які ви хочете включити до зображення. |
| `negative` | `STRING` | Умови, що описують атрибути, які ви хочете виключити з зображення. |
| `latent_image` | `LATENT` | Латентне зображення для зменшення шуму. |
| `seed` | `INT` | Випадковий параметр, що використовується для створення шуму. |
| `steps` | `INT` | Кількість кроків, що використовуються в процесі зменшення шуму. |
| `cfg` | `FLOAT` | Масштаб без класифікаційної безпеки (Classifier-Free Guidance) балансує креативність і відповідність підказці. Вищі значення призводять до зображень, що більш точно відповідають підказці, однак занадто високі значення негативно впливають на якість. |
| `sampler_name` | `COMBO` | Алгоритм, що використовується під час вибору, може впливати на якість, швидкість і стиль отриманого результату. |
| `scheduler` | `COMBO` | Планувальник контролює, як поступово видаляється шум для формування зображення. |
| `denoise` | `FLOAT` | Кількість зменшення шуму, менші значення зберігають структуру початкового зображення, що дозволяє здійснювати зображення-до-зображення. |
| `filename_prefix` | `STRING` | Префікс для файлу збереження. Може включати інформацію форматування, наприклад %date:yyyy-MM-dd% або %Empty Latent Image.width%, щоб включити значення з вузлів. |

### Вихідні дані

| Назва | Тип | Опис |
| --- | --- | --- |
| `image` | `IMAGE` | Декодоване зображення. |
