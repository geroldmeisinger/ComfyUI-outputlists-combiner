## JSON OutputList

![JSON OutputList](JSONOutputList/JSONOutputList.png)

(ComfyUI ნამუშევრის მითითებულია)

შექმნის OutputList-ს JSON ობიექტებიდან მასივების ან დიქტის გამომოძავნით.
გამოიყენებს JSONPath სინტაქსს მნიშვნელობების გამოსამოძავნელად, იხ. [JSONPath on Wikipedia](https://en.wikipedia.org/wiki/JSONPath) .
ყველა დამთხვევის მნიშვნელობა შედგენილია ერთ გრძელ სიაში.
ასევე შეიძლება გამოიყენოთ ეს ნოდი ლიტერალური სტრიქონებისგან ობიექტების შესაქმნელად, მაგალითად `[1, 2, 3]`.
`key`, `value`, `int` და `float` იყენებს `is_output_list=True` (მიუთითებს სიმბოლოთი `𝌠`) და იქნება დამუშავებული შესაბამისი ნოდების მიერ მიმდევრობით.

### შეყვანები

| სახელი | ტიპი | აღწერა |
| --- | --- | --- |
| `jsonpath` | `STRING` | JSONPath გამოყენებულია მნიშვნელობების გამოსამოძავნელად. |
| `json` | `STRING` | JSON სტრიქონი, რომელიც მორგებულია ობიექტად. |
| `obj` | `*` | (არასავალდებულებო) ობიექტი ნებისმიერი ტიპის, რომელიც ჩანაცვლებს JSON სტრიქონს |

### გამოტანები

| სახელი | ტიპი | აღწერა |
| --- | --- | --- |
| `key` | `STRING 𝌠` | საკვანძო სიტყვა დიქტებისთვის ან ინდექსი მასივებისთვის (სტრიქონად). ტექნიკურად ეს არის გლობალური ინდექსი შედგენილი სიისთვის ყველა არასაკვანძო მნიშვნელობისთვის. |
| `value` | `STRING 𝌠` | მნიშვნელობა სტრიქონად. |
| `int` | `INT 𝌠` | მნიშვნელობა მთელი რიცხვის სახით (თუ არ შეიძლება რიცხვის დამუშავება, ნაგულისხმევი მნიშვნელობაა 0). |
| `float` | `FLOAT 𝌠` | მნიშვნელობა ათწილადის სახით (თუ არ შეიძლება რიცხვის დამუშავება, ნაგულისხმევი მნიშვნელობაა 0). |
| `count` | `INT` | სიის სრული რიცხვი ერთი სიაში |
| `debug` | `STRING` | დამუშავებული მნიშვნელობების საფუძე სტრიქონის სახით JSON ფორმატში |
